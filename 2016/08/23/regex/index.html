正则表达式

一小时保你学会正则表达式.

当我们处理大段文章的时候，需要匹配或者替换一些内容的时候，或者需要截取下载页面中你需要的信息的时候，我们就需要用正则表达式去匹配我们需要的东西。并且现在python的网络爬虫的时候，通常要借助正则表达式去匹配元素。所以学会正则表达式也是很有帮助的。

必要的准备

  • 建议安装一个[RegexBuddy]，对于创建正则表达式非常方便。本教程也是使用的该工具
  • 熟悉一下java中的 [Pattern]

好了，接下来就进入正题吧！

1.基本字符(无意义的字符)

所谓基本字符，其实就是没有意义的字符。也就是说，你写出来是什么，就匹配什么。

所包含的字符

  • 大小写字母
  • 数字
  • 汉字
  • 有含义的字符加上“\”(反斜杠)进行转义，比如：\.,\\等

注意

各个字符之间没有特殊符号的时候，他们之间是并的关系。即ab匹配的是文本中的ab，而不文本中所有的是a和b

实例

  • 数字匹配
  • 字母匹配

  • 特殊字符匹配(如果需要匹配的字符是特殊字符，则需要将特殊字符进行转义，即加上”\”)

    介绍几个常见的特殊字符
    \n -> 换行符
    \t -> 制表符(Tab)
    .;+;?;^ -> 正则表达式中的特殊字符(后面都会讲到)

2.标准字符集合

注意:要区分大小写，大小写的意义是相反的。空白符包括空格、换行符、制表符等









\d表示0-9中任何一个数字
\D表示除数字外任何一个字符
\w表示A-Z,a-z,0-9,中任何一个字符
\W表示除A-Z,a-z,0-9,中任何一个字符
\s表示空白符中任何一个字符
\S表示除空白符中任何一个字符
. 表示除换行符外的任何一个字符

实例

  • \d 和 \D

3.自定义字符集合

其基本用法就是将需要匹配的字符用[]括起来

注意

  • ^符号在[]中表示取反的意思
  • -中划线在括号内为到的意思，比如一到九就是1-9
  • 括号中任何正则表达式特殊字符都失去其特殊含义，除了^和-
  • 括号内各个字符是通过或连接的，比如[ab]匹配的是文本中a或者b字符，与单独的ab是不同的。

实例

4.量词

修饰匹配的次数

注意

  • 匹配时，默认的是贪婪模式，即按照匹配最大的字符串进行匹配。如果要是想按非贪婪模式的话，在量词后加?即可
  • 量词后的?和?是不同的~
  • +，*，？默认也是贪婪模式的，但是不能通过？来修饰，转换为非贪婪模式







{m}表达式重复m次
{m,n}表达式重复至少m次，最多n次
{m,}表达式重复至少m次
?表达式重复0次或1次，相当于{0,1}
+表达式重复至少1次，相当于{1,}
*表达式重复至少0次，相当于{0,}

5.字符边界

之前的所有操作都是去匹配一个字符，而接下来要讲的这个内容是匹配位置，即零宽匹配。





^匹配字符串开始的位置
$匹配字符串结束的位置
\b匹配一个单词边界(即前面的字符和后面的字符不全是\w)

6.选择符和分组






|匹配组两边是或的关系，即匹配左边的表达式或者匹配右边的表达式
()捕获组
  • 括号内为一个整体的表达式，与其他的表达式分隔开。
  • 括号内匹配到的元素会自动缓存到内存当中。
  • 匹配到的每一组根据()的位置从1开始进行编号
(?:表达式)如果()的作用仅是与其他表达式隔离，而无需将匹配的结果进行保存，就用该方式
()\n
  • 反向引用
  • 将指定捕获组的字符串再匹配一次
  • 该表达式中的n为具体的捕获组编号

实例

  • 实例一(匹配重复2-4次的数字或者重复1-2次的空白符)
  • 实例二(匹配类似mymy的字符)

注意

在实例二中，(\w{2})匹配到的是重复两次出现的字母、数字或者下划线。
\1代表的是(\w{2})匹配到的内容再匹配一次

7.预搜索

注意

  • 此种匹配仍然是位置匹配，即匹配的只是位置，而不是字符





(?=exp)所匹配表达式的后面满足exp条件
(?<=exp)所匹配表达式的前面满足exp条件
(?!exp)所匹配表达式的后面不满足exp条件
(?<!exp所匹配表达式的前面满足exp条件

实例

  • 实例一(匹配三位数字，且这三位数字后面一位是空白符)
  • 实例二(匹配三位数字，且这三位数字前面一位是中划线)

练习

练习 1 —- 电话号码匹配

  • 条件

    • 电话号码由数字和中划线-组成
    • 电话号码由区号和电话号码组成，中划线左侧是区号，右侧是电话号码
    • 电话号码为7位或者8位
    • 区号为3位或4位，首位是0
    • 移动电话为11位
    • 移动电话前两位为13,15,17,18

  • 正则表达式

    • 电话号码: 0\d{2,3}-\d{7,8}(?=\s)
    • 移动电话: 1[3578]\d{9}(?=\s)
    • 总体: (0\d{2,3}-\d{7,8}(?=\s))|(1[3578]\d{9}(?=\s))

  • 实例

练习 2 —- 邮箱匹配

  • 条件

    • 由数字、字母、@、下划线组成
    • @前由字母和数字组成
    • @后为域名，由字母和.组成

  • 正则表达式

    • @前: [\w-]+
    • @后: [\da-zA-Z]+(.[\da-zA-Z]+){1,2}
    • 总体: [\w-]+@[\da-zA-Z]+(.[\da-zA-Z]+){1,2}

  • 实例
